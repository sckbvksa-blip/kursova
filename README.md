# Курсовий проєкт: Застосунок для управління рецептами (RecipesApp).

## 1. Призначення програми
RecipesApp - це консольний застосунок, створений для зручного управління кулінарними рецептами. Програма дозволяє користувачам зберігати власні рецепти, шукати їх, змінювати пропорції інгредієнтів залежно від потрібної кількості порцій, а також автоматично генерувати списки покупок для обраних страв.

## 2. Короткий опис предметної області
Предметна область проєкту - кулінарія та організація домашнього харчування. Основні сутності системи:
* Рецепт (Recipe): базова одиниця, що містить назву, категорію, час приготування, рівень складності, кроки приготування та статус "Улюблений".
* Інгредієнт (Ingredient): складова частина рецепта. Має назву та точну кількість.
* Кількість та Одиниці виміру (Quantity, MeasurementUnit): визначають об'єм, вагу або кількість інгредієнта (грами, літри, штуки тощо).
* Список покупок (ShoppingList): автоматично згенерований перелік продуктів, необхідних для приготування обраного рецепта.

## 3. Основні можливості системи
Система надає користувачу інтерактивне консольне меню з наступним функціоналом:
* Перегляд усіх збережених рецептів у книзі.
* Додавання нового рецепта (із введенням назви, опису, інгредієнтів та їхніх пропорцій).
* Пошук рецепта за ключовим словом (у назві або описі).
* Масштабування порцій рецепта (перерахунок кількості всіх інгредієнтів на заданий коефіцієнт, наприклад, x2 або x0.5).
* Додавання або видалення рецепта зі списку "Улюблених" (Favorite).
* Автоматична генерація та збереження списку покупок на основі обраного рецепта.
* Видалення рецепта з бази даних.

## 4. Інструкція зі збирання та запуску
Для роботи програми необхідний встановлений .NET 8.0 SDK.

1. Відкрийте термінал у кореневій папці проєкту.
2. Перейдіть до папки основного застосунку, ввівши команду: cd task
3. Виконайте збирання проєкту командою: dotnet build
4. Запустіть програму командою: dotnet run


## 5. Інструкція з використання
Після запуску програми у консолі з'явиться Головне меню:

* Навігація здійснюється шляхом введення номера потрібного пункту (від 1 до 7) та натискання клавіші Enter.
* Для виходу з програми оберіть пункт 0.
* Під час додавання нових інгредієнтів програма попросить ввести одиниці виміру англійською (наприклад: Gram, Liter, Piece). Якщо введено невідоме значення, за замовчуванням встановиться Gram.
* Десяткові дроби (при масштабуванні або введенні ваги) можна вводити як через крапку, так і через кому - програма автоматично їх обробить.

## 6. Формат і спосіб збереження даних
Дані зберігаються локально у текстових файлах формату JSON. 
Система використовує патерн Repository (JsonRepository), який відповідає за серіалізацію та десеріалізацію C#-об'єктів за допомогою стандартної бібліотеки System.Text.Json. Файли зберігаються у структурованому вигляді з відступами (WriteIndented = true) для зручності ручного читання та редагування.

## 7. Перелік і призначення підготовлених файлів із прикладами даних
Програма автоматично створює папку Data у директорії виконання (поруч із .exe або .dll файлом) і працює з такими файлами:

* Data/recipes.json - містить базу даних рецептів.
* Data/shopping_lists.json - містить згенеровані списки покупок.

## 8. Основні модулі, компоненти та їхня відповідальність
Проєкт розділено на логічні простори імен (наближено до Clean Architecture):
* RecipesApp.Core.Domain - ядро (Domain Layer). Містить сутності та структури даних (Recipe, Ingredient, Quantity, ShoppingList), які не залежать від інших модулів.
* RecipesApp.Core.Interfaces - абстракції. Містить інтерфейси IRepository, IRecipeService, IShoppingListService для інверсії залежностей.
* RecipesApp.Core.DataAccess - шар доступу до даних. Містить JsonRepository, який відповідає за запис та читання JSON-файлів.
* RecipesApp.Core.Services - шар бізнес-логіки. RecipeService відповідає за пошук і розрахунки (масштабування), ShoppingListService - за логіку списків покупок.
* RecipesApp.ConsoleUI - шар представлення (User Interface). Відповідає за введення/виведення даних у консоль (Menu, Printer, InputHandler).
* RecipeApp.Tests - окремий проєкт із NUnit-тестами, який містить FakeRepository та покриває бізнес-логіку сервісів (позитивні, негативні та граничні сценарії).

## 9. Конфігурація проєкту
Застосунок не потребує складних файлів конфігурації. Основне налаштування виконується на старті у файлі Program.cs:

* Встановлюється кодування UTF-8 для коректного відображення кирилиці в консолі: Console.InputEncoding = Encoding.UTF8; Console.OutputEncoding = Encoding.UTF8;
* Налаштовуються шляхи до файлів recipes.json та shopping_lists.json під час ініціалізації екземплярів репозиторіїв.
